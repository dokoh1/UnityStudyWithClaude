## Rendering Pipeline이란?

3D 씬을 화면의 2D 픽셀로 변환하는 전체 과정을 관리하는 시스템. "조명을 어떻게 계산할지, 그림자를 어떻게 드리울지, 후처리를 어떻게 적용할지" 등 렌더링의 모든 단계를 정의한다.

Unity는 2018년 이전까지 Built-in Render Pipeline 하나만 제공했지만, 2018년 SRP(Scriptable Render Pipeline)를 도입해 URP와 HDRP를 만들었다.

---

## 1. Built-in Render Pipeline

Unity의 전통적인 렌더링 시스템. 2018년 이전 기본.

**특징**:
- 가장 범용적, 안정적
- 거의 모든 에셋스토어 셰이더/에셋과 호환
- Forward / Deferred 선택 가능
- **SRP가 아니라서 SRP Batcher 사용 불가**

**제약**:
- 커스터마이징 어려움 (C++ 코어)
- 신규 기능이 거의 추가되지 않음 (사실상 레거시)

**적합**: 레거시 프로젝트, 에셋스토어 의존 큰 경우

---

## 2. URP (Universal Render Pipeline)

**"모든 플랫폼에서 잘 작동하는 균형 잡힌 파이프라인"**

**특징**:
- SRP 기반 → SRP Batcher 자동
- Forward+ 렌더링 기본
- Shader Graph 통합
- 2D Renderer 내장 (2D 게임에 특화)
- VR/AR 지원 우수
- 모바일 최적화

**성능**: Built-in보다 10~50% 빠름, 모바일에서 안정적

**그래픽 품질**: 중상급 — AAA급 사실적은 어렵지만 Stylized/Cartoon/Pixel Art 등에 최적

**적합**:
- **모바일 게임**
- 멀티 플랫폼 (PC + 모바일 + 콘솔)
- VR 게임
- 2D 게임
- 인디 게임
- 실사가 아닌 스타일

**실무 예시**: 대부분의 모바일 RPG, 캐주얼 게임, 인디 게임, Genshin Impact(커스텀 많음)

---

## 3. HDRP (High Definition Render Pipeline)

**"고사양 하드웨어에서 사실적 그래픽을 추구하는 파이프라인"**

**특징**:
- SRP 기반
- 물리 기반 렌더링(PBR) 완전 구현
- Volumetric Lighting/Cloud
- **Ray Tracing 지원** (RTX GPU)
- Screen Space Reflection, Screen Space GI
- 고급 포스트 프로세싱 내장
- 물리 기반 카메라

**제약**:
- **모바일 불가**
- 저사양 PC에서 느림
- 빌드 크기 큼
- 학습 곡선 가파름
- PBR 텍스처 필수

**그래픽 품질**: AAA급, 영화/애니메이션 제작 가능

**적합**:
- PC/콘솔 전용 고사양 게임
- AAA급 그래픽 목표
- 건축 시각화 (Archviz)
- 영화/애니메이션 프로덕션
- 자동차 시뮬레이터

## 상세 비교

|  | Built-in | URP | HDRP |
|---|---|---|---|
| 타겟 플랫폼 | 전부 | 전부(모바일 중심) | PC/콘솔 고사양 |
| 그래픽 품질 | 중 | 중~중상 | 최상 (AAA급) |
| 성능 (모바일) | 중 | 높음 | 불가 |
| 성능 (PC 중사양) | 중 | 높음 | 낮음~중 |
| SRP Batcher | ❌ | ✅ | ✅ |
| Shader Graph | △ | ✅ | ✅ |
| Ray Tracing | ❌ | ❌ | ✅ |
| VR 지원 | 중 | 우수 | 제한적 |
| 빌드 크기 | 작음 | 중 | 큼 |
| 학습 곡선 | 낮음 | 중 | 높음 |
| 에셋스토어 호환성 | 최고 | 좋음 | 제한적 |
| 신규 기능 추가 | ❌ | ✅ | ✅ |

---

## 선택 기준

```
플랫폼:
├── 모바일 포함 → URP 확정
├── PC 전용 저/중사양 → URP 또는 Built-in
├── PC/콘솔 고사양 전용 → HDRP 고려
└── VR → URP 강력 권장

그래픽 목표:
├── Stylized/Cartoon/Low-poly → URP
├── Pixel Art/2D → URP (2D Renderer)
├── 사실적이지만 광범위한 플랫폼 → URP
└── 극사실적 AAA급 → HDRP

2026년 기준:
→ 신규 프로젝트는 URP가 기본값
→ 특수 목적이면 HDRP
→ Built-in은 레거시 유지 목적만
```

---

## 전환 시 주의사항

Built-in → URP/HDRP 마이그레이션:
- ⚠️ 셰이더 전부 교체 (보라색 에러)
- ⚠️ 에셋스토어 에셋 호환성 확인
- ⚠️ 포스트 프로세싱 시스템 다름 (PPv2 vs Volume)
- ⚠️ 조명 설정 재작업

→ 초기부터 URP/HDRP로 시작하는 게 훨씬 편함

---

## Forward vs Deferred 렌더링 (추가 지식)

- **Forward**: 오브젝트마다 조명 계산, 라이트 수에 비례해 비용 증가, 투명 처리 쉬움
- **Deferred**: G-Buffer에 저장 후 조명 처리, 많은 라이트에 유리, MSAA 불가
- URP는 Forward+ 기본, HDRP는 Deferred 기본

## 요약 답변 (면접 30초)

> "Unity 렌더링 파이프라인은 Built-in, URP, HDRP 세 가지입니다.
> Built-in은 레거시 시스템이고,
> URP는 모바일부터 콘솔까지 모든 플랫폼에 균형 잡힌 파이프라인,
> HDRP는 PC/콘솔에서 AAA급 사실적 그래픽을 추구하는 파이프라인입니다.
>
> 선택 기준은 명확합니다. 모바일을 타겟팅하면 URP,
> 사실적 그래픽이 필요한 PC/콘솔 전용이면 HDRP,
> 기존 Built-in이면 마이그레이션 비용을 계산해서 결정합니다.
>
> 2026년 기준 신규 프로젝트는 URP가 기본값이며,
> SRP Batcher로 성능 이점이 있고 Shader Graph 통합도 뛰어납니다."

## 오답 노트

1. **주니어 면접에서는 잘 안 나오는 주제** — 파이프라인 선택은 보통 리드/시니어의 결정 영역. 주니어는 "프로젝트에 설정된 파이프라인을 사용" 수준.

2. **Built-in은 사실상 레거시** — Unity가 공식적으로 표현하진 않지만 신규 기능은 URP/HDRP에만 추가되는 추세. 2026년 신규는 URP 기본.

3. **HDRP ≠ "무조건 고화질"** — 고화질이지만 저사양에서 느리고 모바일 불가. 플랫폼이 결정 요소.

4. **SRP Batcher는 URP/HDRP의 핵심 차별점** — Built-in은 못 씀. 성능 차이의 큰 부분이 여기서 옴.

5. **2D 게임도 URP** — URP의 2D Renderer가 2D 전용 라이팅 시스템을 제공.
